задача - отображать parallel и sequential кейсы в разных вариантах, так же изменить типы в интерфейсах

1) paralllel - отображаются как раньше - в дереве (должно быть подписано сверху дерева- сколько parallel кейсов юзаются и тд)
2) sequential кейсы - отображаются таблицей (таблицу надо тоже виртуализировать)

в общем, теперь вместо одного дерева должно располагаться две секции - parallel и sequential

следуй текущему кодстайлу, компоненты для drag n drop можешь посмотреть как реализовано в test case form или create run from cases - CreateForm (dndkit везде)

должны быть реализованы следующие фичи: (запись из постановки)

- Две секции: Sequential (с порядком, плоский список + колонка suite) и Parallel. Секцию без тест-кейсов не отображать.
- Drag & drop для изменения порядка с автосохранением (PATCH запрос)
- Блокировка drag & drop и кнопки редактирования при статусе in_progress (выполняется)
- Перезапуск завершённых кейсов: кнопка «Retest» (как и раньше); доступна по правилам: в Parallel — до завершения Test Run; в Sequential — только после завершения секции или после завершения/остановки Test Run'а.
- Прогресс выполнения по секциям: "Sequential: 3/5 completed, Parallel: 7/10 completed"
